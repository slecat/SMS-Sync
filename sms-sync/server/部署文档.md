# 短信同步服务器部署文档

## 部署环境

- **操作系统**: Ubuntu / Linux
- **端口**: 8004
- **进程管理**: PM2
- **Node.js 版本**: 14.x 或更高

## 部署步骤

### 1. 准备工作

#### 1.1 更新系统
```bash
sudo apt update && sudo apt upgrade -y
```

#### 1.2 安装 Node.js
```bash
curl -fsSL https://deb.nodesource.com/setup_lts.x | sudo -E bash -
sudo apt install -y nodejs

# 验证安装
node -v
npm -v
```

#### 1.2 安装 PM2
```bash
sudo npm install -g pm2

# 验证安装
pm2 --version
```

### 2. 上传代码

将 `sms-sync/server` 目录上传到服务器，例如：
```bash
# 本地压缩代码
cd sms-sync/server
zip -r ../server.zip .

# 上传到服务器
scp ../server.zip user@your-server:/home/user/

# 在服务器上解压
ssh user@your-server
cd /home/user
unzip server.zip
mkdir -p ~/sms-sync-server
mv server/* ~/sms-sync-server/
rm server.zip server
```

### 3. 配置环境变量

#### 3.1 创建 `.env` 文件
```bash
cd ~/sms-sync-server
nano .env
```

#### 3.2 添加配置内容
```env
PORT=8004
```

### 4. 安装依赖

```bash
cd ~/sms-sync-server
npm install --production
```

### 5. 启动服务

#### 5.1 使用 PM2 启动
```bash
pm2 start index.js --name sms-sync-server
```

#### 5.2 查看状态
```bash
pm2 status
pm2 logs sms-sync-server
```

#### 5.3 查看实时日志
```bash
pm2 logs sms-sync-server --lines 100 -f
```

### 6. 配置防火墙

```bash
# Ubuntu 防火墙（UFW）
sudo ufw allow 8004/tcp

# 或使用 iptables
sudo iptables -A INPUT -p tcp --dport 8004 -j ACCEPT
sudo iptables-save > /etc/iptables/rules.v4
```

### 7. 配置开机自启动

```bash
# PM2 保存当前进程列表
pm2 save

# 生成启动脚本
pm2 startup

# 执行显示的命令（通常是这行）
sudo env PATH=$PATH:/usr/bin pm2 startup systemd -u your-username --hp /home/your-username
```

### 8. 配置 Nginx 反向代理（可选）

如果需要使用域名访问，可以配置 Nginx 反向代理：

#### 8.1 安装 Nginx
```bash
sudo apt install -y nginx
```

#### 8.2 创建配置文件
```bash
sudo nano /etc/nginx/sites-available/sms-sync
```

#### 8.3 添加配置内容
```nginx
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://localhost:8004;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_read_timeout 86400;
    }
}
```

#### 8.4 启用配置
```bash
sudo ln -s /etc/nginx/sites-available/sms-sync /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

## PM2 常用命令

### 查看所有进程
```bash
pm2 list
```

### 查看进程状态
```bash
pm2 status sms-sync-server
```

### 查看日志
```bash
# 查看日志
pm2 logs sms-sync-server

# 实时查看日志
pm2 logs sms-sync-server --lines 100 -f

# 清空日志
pm2 flush sms-sync-server
```

### 重启服务
```bash
pm2 restart sms-sync-server
```

### 停止服务
```bash
pm2 stop sms-sync-server
```

### 删除进程
```bash
pm2 delete sms-sync-server
```

## 故障排查

### 1. 端口被占用
```bash
# 查看 8004 端口占用
sudo netstat -tlnp | grep 8004

# 杀死占用进程
sudo kill -9 <PID>
```

### 2. 服务无法启动
```bash
# 查看详细日志
pm2 logs sms-sync-server --err

# 检查 Node.js 版本
node -v
```

### 3. 防火墙问题
```bash
# 检查防火墙状态
sudo ufw status

# 允许 8004 端口
sudo ufw allow 8004/tcp
sudo ufw reload
```

### 4. PM2 服务问题
```bash
# 重启 PM2
pm2 resurrect
pm2 save
```

## 更新部署

当需要更新代码时：

```bash
# 1. 停止服务
pm2 stop sms-sync-server

# 2. 更新代码
cd ~/sms-sync-server
git pull  # 或者重新上传代码

# 3. 重新安装依赖
npm install --production

# 4. 重启服务
pm2 restart sms-sync-server
```

## 安全建议

1. **使用 HTTPS**: 在生产环境中配置 SSL 证书
2. **限制 IP**: 配置防火墙只允许特定 IP 访问
3. **使用 WebSocket Secure (WSS)**: 配置 Nginx SSL 代理
4. **定期更新**: 定期更新 Node.js 和依赖包

## 快速部署脚本

创建 `deploy.sh` 文件：

```bash
#!/bin/bash

# 颜色输出
GREEN='\033[0;32m'
NC='\033[0m' # No Color

echo -e "${GREEN}=== 短信同步服务器快速部署 ===${NC}"

# 1. 进入项目目录
cd ~/sms-sync-server

# 2. 安装依赖
echo -e "${GREEN}安装依赖...${NC}"
npm install --production

# 3. 停止旧进程
echo -e "${GREEN}停止旧进程...${NC}"
pm2 stop sms-sync-server 2>/dev/null || true
pm2 delete sms-sync-server 2>/dev/null || true

# 4. 启动新进程
echo -e "${GREEN}启动服务...${NC}"
pm2 start index.js --name sms-sync-server

# 5. 保存 PM2 配置
echo -e "${GREEN}保存 PM2 配置...${NC}"
pm2 save

# 6. 查看状态
echo -e "${GREEN}服务状态：${NC}"
pm2 status

echo -e "${GREEN}=== 部署完成 ===${NC}"
echo -e "查看日志: pm2 logs sms-sync-server --lines 100 -f"
```

使用脚本：
```bash
chmod +x deploy.sh
./deploy.sh
```
